Your authentication was handled by MagicLink, which validated your token and set a cookie for subsequent requests.

How MagicLink Authentication Works:
  1. You generated a token through the MagicLink dashboard
  2. You accessed a URL with the token as a query parameter
  3. NGINX forwarded the authentication request to the MagicLink service
  4. MagicLink validated the token and returned an authorization header
  5. NGINX's Lua filter set the token as a cookie and redirected to a clean URL
  6. Future requests use the cookie for authentication
